# Build Your Own CLI Tool

Argument parsing, subcommands, config, distribution — production-grade.

## What you'll build

- Parse arguments like git does (subcommands, flags, positional)
- Implement a config file (TOML)
- Colorized, paginated output that behaves with pipes
- Exit codes that work with shell scripts
- Build distributable binaries for macOS/Linux/Windows

## The core idea

A lot of 'engineering' is the last 10% that makes a tool people actually want to use. This track is that 10%.

- Fundamentals
