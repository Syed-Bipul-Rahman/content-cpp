# Build Your Own Search Engine

Inverted index, tokenization, TF-IDF, BM25, query parser.

## What you'll build

- Tokenize and normalize text
- Build an inverted index mapping terms → document lists
- Rank with TF-IDF, then BM25
- Handle boolean queries, phrase queries, proximity
- Benchmark on a real corpus

## The core idea

The same data structure (inverted index) powers Lucene, Elasticsearch, and Google. Once you've written one, you understand all of them.

## Prerequisites

- STL containers
- Hashing
