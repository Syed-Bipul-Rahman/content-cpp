# Build Your Own Shell

A shell is a program that reads commands and runs them. That's it. Every shell
you've used — `bash`, `zsh`, `fish` — started as this exact problem.

## What you'll build

A shell called `msh` that:

- Runs external commands: `ls`, `grep`, `cat`
- Supports pipes: `ls | grep cpp | wc -l`
- Handles redirection: `echo hi > file`, `sort < data.txt`
- Has built-ins: `cd`, `exit`, `export`, `pwd`
- Supports environment variable expansion: `echo $HOME`
- Job control: `&`, `jobs`, `fg`, `bg`
- A reasonable prompt and line editing via `readline`

## The core idea

Every shell, at its core, is a loop:

```
while (true) {
    print prompt
    read line
    parse into commands
    for each command: fork(), then exec() in the child
    parent waits (or doesn't, for &)
}
```

The magic is in three syscalls: `fork()`, `exec()`, and `wait()`. Once you
understand those, the rest is parsing and plumbing.

## Pipes: the trick

A pipe is two file descriptors. One end is write, the other is read. `fork`,
redirect the child's stdout to the write end, exec, then redirect the next
child's stdin to the read end. Chain them. That's `ls | grep cpp`.

```
ls       → writes to pipe write end
grep cpp → reads from pipe read end
```
