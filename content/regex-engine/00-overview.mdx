# Build Your Own Regex Engine

Parse regex, compile to an NFA, run it against input.

## What you'll build

- Tokenize and parse regex syntax into an AST
- Build an NFA using Thompson's construction
- Simulate the NFA against input
- Convert NFA → DFA for speed
- Support `.`, `*`, `+`, `?`, `|`, `()`, `[]`, and anchors

## The core idea

A regex engine is a **compiler** in miniature. You're translating a pattern language into a state machine and running it. This is the track that demystifies compilers for a lot of people.

## Prerequisites

- Data structures (graphs, maps)
- Templates and STL
- Recursion
