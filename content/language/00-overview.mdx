# Build Your Own Programming Language

Lexer → parser → AST → tree-walk interpreter → bytecode VM → GC.

## What you'll build

- Design the language — you decide the syntax and semantics
- Tokenize source text
- Parse into an AST using recursive descent
- Evaluate by walking the tree
- Compile to bytecode and run on a stack VM
- Add a mark-sweep garbage collector

## The core idea

If you've never implemented a language, you don't really know what one is. This track is the cure.

## Prerequisites

- Data structures
- Recursion and tree traversal
- Strong comfort with RAII and ownership
