# Build Your Own Database

A real relational database: pager, B-tree, SQL parser, WAL, transactions.

## What you'll build

- Design an on-disk page format
- Implement a pager with a buffer cache (LRU)
- Build a B+tree for indexed lookups
- Write a SQL lexer and parser (subset: SELECT/INSERT/CREATE TABLE)
- Execute queries via a tree-walking interpreter
- Add a write-ahead log and crash recovery

## The core idea

A database is three hard problems stacked: **persistence** (bytes on disk), **indexing** (finding them fast), and **correctness** (ACID). We'll tackle each in turn.

## Prerequisites

- Pointers and memory layout
- File I/O
- Understanding of the C++ object model
