# Build Your Own Redis

A single-threaded, event-driven key-value store speaking the RESP protocol.

## What you'll build

- Speak the RESP wire protocol
- Handle many connections with a single thread (epoll/kqueue)
- Implement GET, SET, DEL, EXPIRE, INCR, LPUSH, HSET
- Hash table with incremental rehashing
- RDB snapshots and append-only file persistence
- Master-replica replication

## The core idea

Redis taught a generation of engineers that **single-threaded + event-driven** can be faster than multi-threaded done poorly. We'll feel why firsthand.

## Prerequisites

- Networking basics
- Pointers, RAII, move semantics
- Hash tables
